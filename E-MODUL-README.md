# E-Modul Feature Setup Instructions

This document provides the necessary SQL scripts and frontend routing instructions to integrate the new E-Modul feature into your CalMyCare application.

## 1. Supabase SQL Setup

Execute the following SQL queries in your Supabase project's SQL Editor to create the necessary tables, policies, and storage buckets.

### a. Create the `educations` Table

This table will store the E-Modul content.

```sql
-- Create the educations table
CREATE TABLE public.educations (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    title TEXT NOT NULL,
    description TEXT,
    cover_image_url TEXT,
    pdf_url TEXT,
    embed_url TEXT,
    is_published BOOLEAN DEFAULT FALSE NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL,
    updated_at TIMESTAMPTZ DEFAULT NOW() NOT NULL
);

-- Create a function to handle updated_at timestamp
CREATE OR REPLACE FUNCTION public.handle_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- Create a trigger to update the updated_at column on row changes
CREATE TRIGGER on_education_update
BEFORE UPDATE ON public.educations
FOR EACH ROW
EXECUTE PROCEDURE public.handle_updated_at();

-- Enable Row Level Security (RLS) for the educations table
ALTER TABLE public.educations ENABLE ROW LEVEL SECURITY;

-- RLS Policies for educations
-- 1. Allow public read access only to published modules
CREATE POLICY "Allow public read access to published educations"
ON public.educations
FOR SELECT
USING (is_published = true);

-- 2. Allow admin full access (insert, update, delete, select)
CREATE POLICY "Allow admin full access to educations"
ON public.educations
FOR ALL
USING (auth.jwt() ->> 'role' = 'admin')
WITH CHECK (auth.jwt() ->> 'role' = 'admin');
```

### b. Create the `bookmarked_educations` Table

This table tracks which modules a user has saved.

```sql
-- Create the bookmarked_educations table
CREATE TABLE public.bookmarked_educations (
    user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    education_id BIGINT NOT NULL REFERENCES public.educations(id) ON DELETE CASCADE,
    created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL,
    PRIMARY KEY (user_id, education_id)
);

-- Enable RLS for the bookmarked_educations table
ALTER TABLE public.bookmarked_educations ENABLE ROW LEVEL SECURITY;

-- RLS Policies for bookmarked_educations
-- 1. Allow users to view their own bookmarks
CREATE POLICY "Allow users to view their own bookmarks"
ON public.bookmarked_educations
FOR SELECT
USING (auth.uid() = user_id);

-- 2. Allow users to insert their own bookmarks
CREATE POLICY "Allow users to insert their own bookmarks"
ON public.bookmarked_educations
FOR INSERT
WITH CHECK (auth.uid() = user_id);

-- 3. Allow users to delete their own bookmarks
CREATE POLICY "Allow users to delete their own bookmarks"
ON public.bookmarked_educations
FOR DELETE
USING (auth.uid() = user_id);
```

### c. Create Storage Bucket and Policies

This sets up the `e-modules` bucket for storing cover images and PDFs.

```sql
-- Insert the new storage bucket for e-modules
INSERT INTO storage.buckets (id, name, public)
VALUES ('e-modules', 'e-modules', true)
ON CONFLICT (id) DO NOTHING;

-- RLS Policies for e-modules storage bucket
-- 1. Allow public read access to all files in the bucket
CREATE POLICY "Allow public read access to e-modules"
ON storage.objects
FOR SELECT
USING (bucket_id = 'e-modules');

-- 2. Allow admins to upload files to the bucket
CREATE POLICY "Allow admin to insert into e-modules"
ON storage.objects
FOR INSERT
WITH CHECK (bucket_id = 'e-modules' AND auth.jwt() ->> 'role' = 'admin');

-- 3. Allow admins to update files in the bucket
CREATE POLICY "Allow admin to update e-modules"
ON storage.objects
FOR UPDATE
USING (bucket_id = 'e-modules' AND auth.jwt() ->> 'role' = 'admin');

-- 4. Allow admins to delete files from the bucket
CREATE POLICY "Allow admin to delete from e-modules"
ON storage.objects
FOR DELETE
USING (bucket_id = 'e-modules' AND auth.jwt() ->> 'role' = 'admin');
```

## 2. Frontend Routing

Add the following routes to your `src/Router.tsx` file to make the new pages accessible within the app.

```tsx
// src/Router.tsx

// ... other imports
import EModulList from './pages/EModulList';
import EModulAdminList from './pages/admin/EModulAdminList';
import EModulAdminForm from './pages/admin/EModulAdminForm';
import { AdminLayout } from './layouts/AdminLayout'; // Ensure this is imported

// ... inside the <Routes> component

{/* Public E-Modul Route */}
<Route path="/modules" element={<EModulList />} />

{/* Admin E-Modul Routes */}
<Route path="/admin" element={<AdminLayout />}>
  {/* ... other admin routes */}
  <Route path="modules" element={<EModulAdminList />} />
  <Route path="modules/new" element={<EModulAdminForm />} />
  <Route path="modules/:id/edit" element={<EModulAdminForm />} />
</Route>

// ...
```

## 3. Update Navigation

To make the E-Modul pages discoverable, add links to your navigation components.

### a. Admin Sidebar

In `src/components/AdminSidebar.tsx`, add a link to the E-Modul management page:

```tsx
// Example:
<nav>
  {/* ... other links */}
  <Link to="/admin/modules">E-Modul</Link>
</nav>
```

### b. Main Navigation

In `src/components/BottomNavbar.tsx` or `src/components/Header.tsx`, add a link to the public E-Modul list:

```tsx
// Example:
<nav>
  {/* ... other links */}
  <Link to="/modules">E-Modul</Link>
</nav>
```
